Aliran Data (Data Flow)

Setiap pertanyaan pengguna melalui beberapa langkah pemprosesan untuk memastikan jawapan yang diberikan adalah relevan dan tepat.

1
💬

Input Soalan

Pengguna memasukkan soalan dalam bahasa semulajadi melalui antara muka chat

2
🏷️

Klasifikasi Soalan

Soalan diklasifikasikan sebagai dalaman (JKST), luaran, berita JKST, atau hibrid untuk menentukan sumber data

3
🔍

Retrieval Pelbagai Sumber

Dokumen relevan diambil dari Google Cloud RAG, fail pengetahuan tempatan, berita JKST, dan/atau carian web Tavily

4
🔀

Reranking

Hasil carian disusun semula menggunakan Cohere rerank-v3.5 untuk memilih 5 dokumen paling relevan daripada 15 hasil awal

5
🤖

AI Generation

Gemini 2.0 Flash menjana jawapan berdasarkan konteks dokumen yang diambil dan disusun semula

6
✅

Response

Jawapan dipaparkan bersama rujukan dokumen untuk pengesahan. Respons juga di-cache untuk 5 minit

🤖

Integrasi Bot Telegram

Pembantu Pintar JKST kini tersedia di Telegram melalui bot @jkst_ai_bot. Bot ini menggunakan enjin AI yang sama seperti versi web untuk memberikan respons yang konsisten dan berkualiti tinggi.

1
💬

Mesej dari Telegram

Pengguna menghantar soalan melalui chat Telegram

2
📡

Webhook Handler

Telegram API menghantar update ke /api/telegram/webhook

3
⚙️

Pemprosesan Mesej

Backend memproses perintah atau menghantar soalan ke RAG

4
🧠

AI Response

Menggunakan RAG dan Gemini untuk menjana jawapan

5
✉️

Balas ke Telegram

Jawapan dihantar balik kepada pengguna melalui Telegram API

⚙️ Maklumat Teknikal Bot Telegram

  • Nama Bot: @jkst_ai_bot
  • Endpoint Webhook: POST /api/telegram/webhook
  • Perintah Tersedia: /start, /help, /clear
  • Penyimpanan Perbualan: Per-chat in-memory storage (maks 10 pasang mesej)
  • Had Saiz Mesej: 4096 aksara (auto-split)
  • Sokongan Format: HTML untuk penyimpanan format
  • Tipologi Keselamatan: Webhook secret token untuk pengesahan
🎙️

Ciri Suara (Voice Features)

Pembantu Pintar JKST menyokong interaksi suara melalui Speech-to-Text (STT) dan Text-to-Speech (TTS), membolehkan pengguna bertanya secara lisan dan mendengar jawapan.

1
🎤

Rakaman Suara

Pengguna merakam soalan melalui mikrofon pada antara muka web

2
📝

Transkripsi (STT)

Audio dihantar ke OpenAI Whisper API untuk ditukar kepada teks dalam Bahasa Malaysia

3
🤖

Pemprosesan AI

Teks yang ditranskripsi diproses melalui pipeline RAG seperti biasa

4
🔊

Sintesis Suara (TTS)

Jawapan ditukar kepada audio menggunakan Google Cloud TTS (suara ms-MY Wavenet)

⚙️ Maklumat Teknikal Suara

  • STT Engine: OpenAI Whisper (model whisper-1)
  • TTS Engine: Google Cloud Text-to-Speech
  • Bahasa: Bahasa Malaysia (ms-MY)
  • Suara TTS: ms-MY-Wavenet-A (perempuan)
  • Format Audio: WebM (input), MP3 (output)
  • Had Teks TTS: Maksimum 5,000 aksara
  • Endpoint STT: POST /api/voice/transcribe
  • Endpoint TTS: POST /api/voice/synthesize
📰

Pengambilan Berita JKST

Sistem ini boleh mengambil berita dan aktiviti terkini secara langsung dari laman web rasmi JKST (syariah.terengganu.gov.my) apabila pengguna bertanya tentang berita atau aktiviti terkini.

1
❓

Pengesanan Soalan Berita

Sistem mengesan kata kunci seperti "berita terkini", "aktiviti JKST", "program terbaru" melalui Query Classifier

2
🌐

Pengambilan dari Laman Web

Berita diambil secara langsung dari laman web rasmi JKST dan di-cache selama 30 minit

3
📋

Pembentangan

Berita diformat dan disertakan sebagai konteks untuk Gemini menjana jawapan yang terkini
